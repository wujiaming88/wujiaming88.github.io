OpenClaw 2026.3.2 版本更新详解

一、版本概述

OpenClaw 2026.3.2 是一个重要的功能更新版本，包含了大量新功能、改进和 bug 修复。该版本重点提升了多通道支持、安全性、可扩展性和开发者体验。

二、主要新功能

1. 秘密管理增强

  • 扩展了 SecretRef 支持到所有用户提供的凭证表面（共 64 个目标）
  • 新增 openclaw secrets 规划/应用/审计流程
  • 新增入职 SecretInput UX
  • 未解析的引用现在会在活动表面上快速失败，非活动表面则报告非阻塞诊断

2. PDF 工具支持

  • 新增一流的 pdf 工具，支持 Anthropic 和 Google PDF 提供商
  • 为非原生模型提供提取回退
  • 可配置默认值（agents.defaults.pdfModel, pdfMaxBytesMb, pdfMaxPages）
  • 新增专门的 PDF 工具文档页面

3. 多通道广播支持

  • Feishu 新增多代理组广播支持
  • 统一了 Lark SDK 错误处理
  • 修复了权限授予 URL 中的无效范围名称

4. 会话附件支持

  • sessions_spawn 新增内联文件附件支持（仅子代理运行时）
  • 支持 base64/utf8 编码
  • 转录内容编辑
  • 生命周期清理
  • 可配置限制（tools.sessions_spawn.attachments）

5. Telegram 增强

  • 默认启用 Telegram 流式传输（partial 模式）
  • 私人预览流式传输使用 sendMessageDraft
  • 新增 disableAudioPreflight 配置选项，可跳过语音笔记的提及检测预飞行转录

6. 配置验证

  • 新增 openclaw config validate 命令，可在网关启动前验证配置文件
  • 启动无效配置错误中包含详细的无效键路径

7. Ollama 嵌入支持

  • 新增 memorySearch.provider = "ollama" 和 memorySearch.fallback = "ollama" 支持
  • 为内存嵌入请求遵循 models.providers.ollama 设置

8. Zalo 个人插件重构

  • 使用原生 zca-js 集成重构 Zalo 个人插件
  • 移除了外部 CLI 传输依赖
  • 完全在 OpenClaw 内部实现 QR/登录 + 发送/监听流程

三、重要改进

1. 插件 SDK 增强

  • 暴露 channelRuntime 到 ChannelGatewayContext，使外部通道插件可以访问共享运行时助手
  • 暴露 runtime.system.requestHeartbeatNow(...)，使扩展可以在排队系统事件后立即唤醒目标会话
  • 暴露 runtime.events.onAgentEvent 和 runtime.events.onSessionTranscriptUpdate 用于扩展端订阅

2. 媒体理解增强

  • 新增可选 tools.media.audio.echoTranscript + echoFormat，可向原始聊天发送代理前转录确认消息
  • 新增 api.runtime.stt.transcribeAudioFile(...)，使扩展可以通过 OpenClaw 配置的媒体理解音频提供商转录本地音频文件

3. CI/CD 优化

  • 减少检查构建和 Windows 作业的关键路径
  • 使节点依赖安装在设置操作中可选
  • 加速 Windows 依赖预热
  • 将 Windows 测试分片为六分之一并跳过缓存恢复

4. 文档改进

  • 新增专门的 PDF 工具文档页面
  • Feishu 文档添加了 verificationToken 并对齐了 zh-CN 与 EN 版本

四、重大变更

1. 入职默认值变更

  • 新本地安装默认 tools.profile 为 messaging
  • 新设置不再默认包含广泛的编码/系统工具，除非显式配置

2. ACP 调度默认启用

  • ACP 调度现在默认启用，除非显式禁用（acp.dispatch.enabled=false）
  • 如果需要暂停 ACP 转向同时保留 /acp 控制，请设置 acp.dispatch.enabled=false

3. Plugin SDK 变更

  • 移除了 api.registerHttpHandler(...)
  • 插件必须通过 api.registerHttpRoute({ path, auth, match, handler }) 注册显式 HTTP 路由
  • 动态 webhook 生命周期应使用 registerPluginHttpRoute(...)

4. Zalo 个人插件依赖变更

  • Zalo 个人插件不再依赖外部 zca 兼容的 CLI 二进制文件
  • 升级后应使用 openclaw channels login --channel zalouser 刷新会话

五、Bug 修复

1. 稳定性修复

  • 插件命令/运行时强化：在注册边界验证和规范化插件命令名称/描述
  • Telegram：保护重复令牌检查和网关启动令牌规范化，防止 token.trim() 崩溃
  • Discord：在网关已连接时立即推送 connected 状态快照
  • Feishu/LINE：将每组 systemPrompt 配置转发到入站上下文 GroupSystemPrompt

2. 兼容性修复

  • Feishu/Plugin SDK 兼容性：加载 Feishu 监控状态时添加安全的 webhook 默认回退
  • Gateway/Subagent TLS 配对：允许经过身份验证的本地 gateway-client 后端自连接跳过设备配对
  • Browser/CDP 启动诊断：在启动超时错误中包含 Chrome stderr 输出和 Linux no-sandbox 提示

3. 安全性修复

  • Synology Chat/webhook 入口强化：通过共享请求体保护强制有界体读取（大小 + 超时）
  • Voice-call/运行时生命周期：通过重置失败的运行时承诺防止 EADDRINUSE 循环

六、总结

OpenClaw 2026.3.2 版本是一个全面的更新，显著提升了多通道支持、安全性、可扩展性和开发者体验。特别是在秘密管理、PDF 工具、多通道广播和插件 SDK 方面的改进，使得 OpenClaw 更加适合企业级应用和复杂的工作流场景。

建议所有用户尽快升级到该版本，以获得更好的性能、安全性和功能体验。
